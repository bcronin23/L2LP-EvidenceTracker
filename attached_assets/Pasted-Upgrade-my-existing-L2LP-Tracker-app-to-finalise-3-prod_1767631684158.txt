(1) Evidence upload must support files + photos
=====================================
- On the “Upload Evidence” screen, the user must be able to:
  - pick from device files (documents, images, videos)
  - pick photos from gallery
  - take a photo on mobile (camera capture)
- Implement a file input that supports:
  - accept="image/*,video/*,audio/*,.pdf,.doc,.docx,.ppt,.pptx,.xls,.xlsx"
  - mobile camera capture (capture="environment")
  - multiple files optional (v2), but at minimum single-file upload must work reliably
- Show a preview:
  - images: thumbnail
  - video/audio: filename + type + size
  - documents: filename + icon
- Upload to Supabase Storage and store metadata in DB:
  file_url, file_name, file_type, file_size

  - block unknown/unsafe types
- UX:
  - the upload flow should be 2–4 taps: Student → Choose file / Take photo → select outcomes → Save

=====================================
(2) School account / Admin model (multi-staff access)
=====================================
Goal: One school “organisation” owns the data. Admin registers students. All staff sign in with their school email to access the same student portfolios and upload evidence.

Implement multi-tenant structure:
Tables:
- organisations (id, name, allowed_domains text[], created_at)
- organisation_members (id, organisation_id, user_id, role enum: ADMIN|STAFF, created_at)
- students add organisation_id
- evidence add organisation_id
- (learning_outcomes can be global shared OR per org; simplest: global table without org_id)

Auth behaviour:
- Sign up / sign in using Supabase Auth.
- When a user signs up, they must be attached to an organisation.
- Provide an Admin-only screen:
  - Create organisation (name + allowed domain like “kwetb.ie”)
  - Invite staff (generate invite link/code OR add email to allowlist)
- Staff onboarding options (pick one and implement cleanly):
  A) Invite code: admin generates code; staff enters code after sign-up; app links them to org.
  B) Allowed domain auto-join: if email ends with allowed domain, allow join; admin approves in Members screen.

Permissions:
- ADMIN can:
  - create/edit students
  - manage staff members (add/remove, role)
  - import outcomes JSON (optional)
- STAFF can:
  - view students
  - upload evidence
  - view evidence library
  - cannot delete students or remove staff

Row Level Security (RLS):
- Users can read/write only data belonging to their organisation_id via organisation_members table membership.
- Provide complete SQL for tables + enums + policies.
